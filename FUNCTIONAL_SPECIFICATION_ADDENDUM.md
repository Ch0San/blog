# 기능 명세 추가 사항 (2025-11-12)

## 인증/에러 처리 정책(업데이트)
- 비인증 접근 분기: 일반 페이지는 로그인 페이지(`/member/signin`)로 리다이렉트, AJAX·API·좋아요(XHR 포함) 요청은 401(Unauthorized)로 응답한다.
- 401 응답 처리(JS): "로그인이 필요합니다." 알림 후 로그인 페이지로 이동한다.
- 허용 경로: `/error`, `/401`는 인증 없이 접근 허용.
- 에러 페이지 라우팅: 401 → `fragments/index_401`, 403 → `fragments/index_403`, 404 → `fragments/index_404`, 500 → `fragments/index_500`.

## 메시지/배너 및 문구 통일
- 상단 배너: 상세 화면에서 `param.success`/`param.error`가 있으면 `.alert-success`/`.alert-error` 배너를 출력한다.
- 스타일: `style.css`에 `.alert`, `.alert-success`, `.alert-error` 정의(일관된 배경/테두리/텍스트 색상).
- 버튼/문구: 좋아요 토글 버튼 텍스트는 "좋아요"/"좋아요 취소", 401 알림 문구는 "로그인이 필요합니다."로 통일.
- 댓글 성공 메시지(쿼리): 생성/수정/삭제 성공 시 각각 `?success=댓글이 등록/수정/삭제되었습니다.`

## Posts/Stories 오류 처리 UX 정리
- Posts 좋아요: 401 → "로그인이 필요합니다." 알림 후 로그인, 기타 오류 → "좋아요 처리 중 오류가 발생했습니다." 알림.
- Posts 댓글 좋아요: 401 → 로그인 유도, 기타 오류 → "댓글 좋아요 처리 중 오류가 발생했습니다." 알림.
- Stories 좋아요/댓글 좋아요: 위와 동일한 정책 적용.
